Clinch IT — Shopify Build: Prioritised Implementation Timeline

v1.0  ·  June 2026  ·  Companion to Architecture v2.1 and Configuration v1.3

This timeline breaks the build into discrete, assignable actions across three tracks — Design, Configuration, Development — sequenced by dependency. Actions marked Parallel can run alongside other actions in the same phase with no blocking relationship. Actions marked Blocks → name what they unlock. Actions marked Blocked by name their prerequisite. Each action references the source document and section so detail can be looked up directly.

Unblock

Summary

Critical Path

The longest unbroken dependency chain from start to launch:

0.1 VAT → 1.1 Metafields → 1.2/1.3 Taxonomy + Tags → 2.1 Collections → 2.2 Collection Rules → 3.1 Search & Discovery → 3.2 Filter Drawer → 3.3 Collection Templates → 3.4 Product Cards → 3.5 Product Page → 4.2/4.3 Trust Pages → 5.1 Stock Intake → 5.2 Filter Health Check → 5.4–5.8 QA → Launch

Everything else — payments, tax setup beyond VAT, shipping zones, checkout configuration, policies, transactional emails — runs in parallel against this spine and does not extend it, provided:

  • Phase 0 decisions (0.1–0.5) are answered within the first 1–2 days
  • The Phase 4 quick decisions (4.1a–c) are answered before policy/email drafting begins, since 13 email/policy instances and three policy documents wait on them
Two items most likely to silently extend the critical path if delayed:
  • OD-C11 (VAT Margin Scheme) — affects pricing on every SKU; late resolution means re-pricing the entire catalogue post-intake
  • 0.4 (Grade D decision) — touches collections (2.3), filters (3.2), and the grading guide (4.2) simultaneously; late resolution means rework across three areas rather than one
