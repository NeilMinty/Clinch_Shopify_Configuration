How Shopify Organises and Displays Products — A Plain-English Primer

This section is for anyone who hasn't worked inside Shopify before. It explains the four building blocks the rest of this document is built on — what they are, why they exist, and how they connect to each other. If you're already familiar with Shopify, skip ahead to Section 1.

The Problem Shopify's Defaults Don't Solve

Every Shopify product comes with a fixed set of standard fields: title, price, description, images, SKU, and stock level. For most products, that's enough. For refurbished tech, it isn't. A buyer choosing a laptop needs to know the RAM, storage, processor, battery health, and operating system before they'll buy. None of those fields exist in Shopify by default. The four tools below are how you solve that.

1. Metafields — Structured Product Data

A metafield is a custom data field you attach to a product. Think of it as adding an extra column to a spreadsheet that Shopify doesn't include by default.

For a refurbished MacBook Air M1, you would create metafields for ram_gb, storage_gb, battery_health_pct, operating_system, and so on. Each metafield has a type — integer, text, boolean (yes/no), decimal — which determines what kind of value it can hold and how Shopify can use it.

Metafields are set up once as definitions (the field template), then filled in per product (the value). Once defined, they appear in the product editor so whoever is entering stock can fill them in on intake.

Where to set them up: Shopify admin → Settings → Custom data → Metafields → Products → Add definition. Name the field, choose its type, and save. It will then appear on every product page in the Metafields section.

2. Tags — Simple Labels for Routing

A tag is a plain text label applied to a product. Unlike metafields, tags have no type or structure — they're just words. brand-apple, gaming, grade-a, os-windows.

Tags are best used for routing products into collections and for internal filtering in the Shopify admin. They are fast to apply (including in bulk) and easy to query with collection rules.

The important distinction from metafields: tags are blunt instruments. They tell Shopify a product belongs to a group. Metafields tell Shopify specific facts about a product. You use both — tags to sort products into the right places, metafields to display the right information and power the right filters.

Where to set them up: Tags are added directly on the product page in the Tags field. No prior setup required. Consistency matters — brand-apple and Brand Apple are different tags to Shopify, so agree the format before listing stock.

3. Automated Collections — Rules-Based Product Grouping

A collection is a group of products. Shopify has two types: manual (you add products by hand) and automated (Shopify adds products automatically when they match rules you define).

For a refurb store with high product turnover, automated collections are essential. You define the rule once — "Product type is Laptop AND vendor is Apple" — and every Apple laptop added to Shopify appears in the Apple Laptops collection automatically, and disappears when it sells. No manual maintenance.

Collection rules can use standard product fields (vendor, type, price, tags) as well as metafields. Using a metafield as a collection condition requires one extra step: enabling the "Smart collections" option on that metafield definition before it becomes available as a rule.

Enabling a metafield for collection rules: Shopify admin → Settings → Custom data → Metafields → Products → click the metafield definition → Options section → enable Smart collections → Save. The metafield then appears as a condition option when creating or editing an automated collection.
Tags vs metafields for collection rules: Both work. Tags are simpler to set up and faster to apply in bulk. Metafields are more structured and less prone to typos. The recommendation in this document is to use both — tags for broad routing (brand, type, grade), metafields for anything that needs precision (condition grade used as a collection condition and a filter and a display field).

4. Search & Discovery — Connecting Data to Filters

Search & Discovery is a free Shopify app that powers the filter panel customers see on collection pages — the checkboxes and price sliders that let someone narrow "All Laptops" down to "Apple, 16GB RAM, under £400, Grade A".

By default it only filters by price and availability. To add filters for RAM, storage, condition grade, and everything else, you connect it to your metafield definitions. Each metafield you want to use as a filter needs two things: storefront access enabled on the definition, and a filter entry created in the app.

Step 1 — Enable storefront access on the metafield: Shopify admin → Settings → Custom data → Metafields → Products → click the definition → Options section → check Storefront → Save.

Step 2 — Add it as a filter: Shopify admin → Apps → Search & Discovery → Filters → Add filter → select your metafield definition as the source → set the display label (what customers see) → Save.
Default Shopify filter behaviour: Price renders as a range slider natively. All other metafield filters render as checkbox lists — one checkbox per discrete stored value. This means you cannot use a raw integer metafield (like battery health percentage) as a range-based filter in default Shopify. For range-style filters such as battery health bands (90%+, 80–90%) and screen size bands (13–14"), this build uses tags applied on intake instead. The raw metafield value is still stored and shown on the product page — it just doesn't drive the filter UI directly.

How It All Joins Up — A Worked Example

Here is the complete journey for a single unit of stock: an Apple MacBook Air M1, 16GB RAM, 256GB SSD, Grade A, battery health 91%.

StepActionWhereResult
1. Create product Add the MacBook Air as a new product. Set vendor to Apple, product type to Laptop. Shopify admin → Products → Add product Product exists in Shopify.
2. Add tags Apply tags: brand-apple, grade-a, os-macos Product page → Tags field Product is routed into the Apple Laptops collection and Grade A collection automatically.
3. Fill in metafields Set ram_gb = 16, storage_gb = 256, condition_grade = Grade A, battery_health_pct = 91, operating_system = macOS, condition_notes = "Light marks on base. Full working order. Charger included." Product page → Metafields section Structured data is stored against this unit.
4. Customer browses Customer visits /collections/apple-laptops and sets filters: RAM = 16GB, Condition = Grade A. Storefront filter panel Search & Discovery reads the metafield values and surfaces only matching products. This MacBook appears.
5. Customer views product Customer clicks through to the product page. Product page The theme reads the metafield values and displays them in the spec table: RAM, storage, OS, battery health, condition notes.
6. Product sells Customer buys. Shopify sets inventory to 0. Automatic Product is removed from all collections and the eBay sync sets the listing quantity to 0.

Filter Architecture

Implemented via the Shopify Search & Discovery app (free, no third-party filter app). This build targets default Shopify UI behaviour throughout.

Default Shopify filter behaviour. Price is a native slider. Metafield filters render as checkbox lists — one checkbox per discrete value. This means range-based filters (battery health bands, screen size bands) cannot be driven by raw integer or decimal metafields in default Shopify. Instead, these are handled via tags applied on intake (e.g. battery-90-plus, screen-13-14). The underlying metafield values are still stored and displayed on the product page — they just don't drive the filter UI.

Implementation Notes

  • This build uses default Shopify + Search & Discovery only. No third-party filter app.
  • All filterable metafields must have Storefront enabled in Settings → Custom data → Metafields → Products before they can be added as filters in Search & Discovery
  • Price is a native Shopify filter and renders as a slider automatically
  • Metafield filters render as checkbox lists in default Shopify — one checkbox per discrete stored value
  • Battery health and screen size use tags for filtering (bucket values set on intake) rather than raw metafield values, because range-based checkbox filters cannot be driven by integer/decimal metafields in default Shopify
  • Mobile: filters open as a full-screen drawer with "Apply" CTA at bottom
  • "Clear all" appears whenever any filter is active
  • Only enable filters with sufficient stock density at launch — add others as inventory grows

Product Variant Strategy

Variants are for true buying choices only — things the customer selects at purchase. Spec and condition differences between units are separate products.

ScenarioApproachReason
Same model, different colour, identical specVariants ✓Genuine buyer choice
Same model, same condition, different storage — multiple units in stock simultaneouslyVariants ✓Genuine buyer choice
Same model, different RAMSeparate productsDifferent unit, different price
Same model, different condition gradeSeparate productsHonest per-unit condition representation
Default for refurb stock: Each physical unit = one product. Given shallow stock depth per SKU, variant complexity is rarely warranted.
